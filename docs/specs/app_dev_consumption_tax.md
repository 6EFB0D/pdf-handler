# 個人開発者のアプリ販売における消費税の取り扱い

> 作成日：2026年3月22日  
> ※ 本資料は一般的な情報提供を目的としたものです。個別の税務判断は税理士・税務署にご相談ください。

---

## 1. 基本：免税事業者 vs 課税事業者

### 課税事業者になる条件

| 判定基準 | 内容 |
|---|---|
| **基準期間**（前々年）の課税売上高 | 1,000万円超 → 課税事業者 |
| **特定期間**（前年1月〜6月）の課税売上高 | 1,000万円超 → 課税事業者になる可能性あり |
| **特定期間の給与等支払額** | 1,000万円超（売上高と両方超えた場合のみ） |

- スタート時点（売上なし）は**自動的に免税事業者**となる
- 開業1年目・2年目は前々年が存在しないため、原則として免税事業者

---

## 2. 「給与等支払額」の正確な意味

**よくある誤解：自分の給与収入が1,000万円を超えると課税事業者になる → ❌ 誤り**

### 「給与等支払額」に含まれるもの

| 対象 | 該当するか |
|---|---|
| 自分が会社から受け取る給与収入 | ❌ 含まれない |
| 自分の事業からの生活費（事業主報酬） | ❌ 含まれない |
| **従業員・専従者への給与・賞与** | ✅ 含まれる |
| 委託先（外注先）への支払い | ❌ 「外注費」扱いのため含まれない |

### 個人アプリ開発者への影響

- 従業員なし・委託先なしの場合、**「給与等支払額」= ゼロ**
- 特定期間の判定は実質的に**課税売上高だけで判断**すれば OK
- 課税売上高が1,000万円を超えても、給与等がゼロなら**給与等基準で免税と主張できる**（選択適用可）

---

## 3. App Store / Google Play 経由の販売

### 2025年4月〜 プラットフォーム課税の開始

2025年4月1日より、**Apple・Googleが消費税を代理で申告・納付**する制度が開始。

- 国外デベロッパーのアプリに対する消費税は、Apple/Google側が処理
- **インボイス登録していない小規模開発者のアプリ**でも、プラットフォームからインボイスが発行される
- 個人開発者が直接消費税を納付する必要はない（売上が1,000万円超の課税事業者の場合は別途申告が必要）

---

## 4. インボイス制度との関係

### 対応が必要なケース / 不要なケース

| 販売先 | インボイス対応の必要性 |
|---|---|
| **一般消費者（個人）向け** | ❌ 不要（買い手は仕入税額控除を必要としない） |
| **法人・事業者向け（B2B）** | ⚠️ 取引先がインボイスを求める場合あり |
| 取引先が簡易課税制度を選択している場合 | ❌ 不要（インボイスなしでも仕入税額控除できる） |

---

## 5. 課税事業者になった場合の軽減措置

### 2割特例（→ 3割特例へ延長）

免税事業者からインボイス登録事業者になった場合に使える特例。

| 期間 | 特例内容 |
|---|---|
| 〜2026年度 | **2割特例**：売上税額の2割（売上高の約2%）を納税額とする |
| 2027〜2028年度 | **3割特例**：個人事業者に限り3割に変更して2年間延長 |

- 事前の届出・手続き不要
- 確定申告時にe-TAXまたは申告書上で選択するだけ
- 基準期間の課税売上高が1,000万円以下の人が対象

---

## 6. 消費税の申告・納税時期

- **課税期間**：1月1日 〜 12月31日（個人事業主）
- **申告・納税期限**：翌年3月31日まで（振替納税の場合は約1ヶ月延長）

---

## 7. 価格設定と請求書の書き方

### 免税事業者でも消費税込みの価格設定は可能

国税庁の公式見解：「免税事業者であっても、仕入れの際に負担した消費税相当額を取引価格に上乗せして請求することは適正な転嫁として、何ら問題はない」

### 販売先別の推奨表示方法

| 販売先 | 推奨する価格表示 |
|---|---|
| **一般消費者（Stripe等）** | 税込価格のみ表示（総額表示義務あり） |
| **法人・事業者（請求書・銀行振込）** | 税込価格のみ表示 or「消費税**相当額**」として別表記 |

### 請求書の書き方ポイント

| 書き方 | 説明 |
|---|---|
| ✅ **税込価格のみ表示**（最推奨） | 「10,000円（税込）」とだけ記載。最もシンプルで誤解がない |
| ✅ **消費税相当額として別表記** | 「本体9,091円 ＋ 消費税相当額909円」と記載 |
| ❌ ~~「消費税」として別表記~~ | インボイスと誤認されるリスクあり。避けた方が無難 |
| ❌ ~~登録番号（T＋13桁）を記載~~ | インボイス登録していない場合は記載禁止（罰則あり） |

### Stripe 利用時の実務

- 税率設定機能はあるが、**免税事業者の場合は「税込」の単一価格として設定するのが最もシンプル**
- 例：月額1,100円（消費税込み）として設定 → そのまま1,100円で請求

---

## 8. 今後の判断フロー（個人アプリ開発者向け）

```
スタート
 └─ 売上 < 1,000万円（免税事業者）
     ├─ 一般消費者向け販売のみ → インボイス登録不要
     │    税込価格で設定・請求するだけでOK
     │
     └─ 法人・事業者向け販売あり
          ├─ 取引先がインボイスを求めない → 登録不要
          └─ 取引先がインボイスを求める → インボイス登録を検討
               └─ 登録した場合、2割特例（→3割特例）を活用
```

## 9. PDF Handler への適用

### 価格設定（v1.1.0〜）

| 項目 | 内容 |
|------|------|
| **価格** | ¥4,980（消費税不課税） |
| **表示** | 「¥4,980（消費税不課税）」と明示 |
| **Stripe** | 不課税の単一価格として設定（税率設定は使わない） |
| **領収証・請求書** | 税込価格のみ表示、または「消費税相当額」として別表記。登録番号は記載しない |

## 参考：重要な用語

| 用語 | 説明 |
|---|---|
| **免税事業者** | 課税売上高1,000万円以下で消費税納税が免除される事業者 |
| **課税事業者** | 消費税を申告・納税する義務がある事業者 |
| **インボイス（適格請求書）** | 課税事業者のみが発行できる、登録番号入りの請求書 |
| **仕入税額控除** | 売上の消費税から仕入の消費税を差し引く仕組み |
| **2割特例（3割特例）** | 免税→課税転換時の消費税負担を軽減する経過措置 |
| **プラットフォーム課税** | App Store / Google Play等が消費税を代理納税する制度（2025年4月〜） |
| **総額表示義務** | 消費者向けの価格は税込総額で表示しなければならないルール |
