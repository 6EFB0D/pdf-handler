# ライセンスコード仕様書

## 1. 形式

### 1.1 コンパクト形式（v1.2〜・現行の新規発行）

**記号として 32 文字**（区切りハイフンは含めない）。  
**Supabase の `license_key`・メール・画面共有の正準形**は、ダッシュボードやコピー時の可読性のため **4 文字ごとにハイフン**を入れたものとする（**32 記号 + 7 ハイフン = 39 文字**）。

```
PDFH-P101-0123-4567-89AB-CDEF-GHJK-MNPQ   （例。実際の記号は Crockford Base32）
```

- **構成（ハイフン除く）**: `app_id` 4 文字（`PDFH` / `ZIPS` / `PICT`）+ 形態 4 文字（例 `P101`）+ **Crockford Base32 で 24 文字**（内部 15 バイト = 乱数 11 + HMAC-SHA256 先頭 4 バイト）
- **署名対象**: `{app_id}:{形態}:{11バイトをHEX22桁大文字}` を `LICENSE_SECRET_KEY` で HMAC-SHA256
- **入力**: 区切り付き・32 記号の連続どちらも正規化で同一キーとして照合

### 1.2 長形式（v1.1〜・HMAC 付き・既存キー）

```
PDFH-<ライセンス形態4文字>-<シリアル部28文字>-<HMAC署名（16進）>
```

- **HMAC署名**: 署名対象 `{app_id}:P101:{シリアル部28桁hex}` を HMAC-SHA256（環境変数 `LICENSE_SECRET_KEY`）

**内部形式の例**: `PDFH-P101-A1B2C3D4E5F6G7H8I9J0K1L2M3-1A2B3C4D`

### 1.3 長形式の表示（4桁区切り・シリアル部のみ）

メール・UI など、長形式のシリアル部を読みやすくする場合:

```
PDFH-P101-A1B2-C3D4-E5F6-G7H8-I9J0-K1L2-M3-…-HMAC
```

## 2. ライセンス形態（4文字）

### 2.1 種別（先頭2文字）

| コード | 意味 |
|--------|------|
| P1 | 買い切り（Purchased）v1 |
| P2 | 買い切り v2（将来用） |

### 2.2 バージョンコード（後ろ2文字）

| コード | 意味 |
|--------|------|
| 01 | メジャーバージョン 1（買い切り v1.x 用） |
| 02 | メジャーバージョン 2（買い切り v2.x 用、将来） |

### 2.3 形態コード一覧

| 4文字 | 種別 | バージョン制限 |
|-------|------|----------------|
| P101 | 買い切り v1 | v1.x まで |
| P102 | 買い切り v2 | v2.x まで（将来） |

> **注**: `S1xx` / `S2xx` などのコードは過去の設計メモで予約されていたものです。本製品（未公開・買い切りのみ）では扱いません。

---

## 3. シリアル部（28文字）

- **文字種**: 0-9, A-F（16進数大文字）
- **生成**: 暗号論的に安全な乱数から生成
- **一意性**: 重複しないこと（UUID ベースまたは十分なエントロピー）

---

## 4. バリデーション

### 4.1 形式チェック（新形式：HMAC付き）

- 正規表現: `^PDFH-P[12][0-9]{2}-[0-9A-F]{28}-[0-9A-F]+$`
- シリアル部・HMAC部はハイフン除去して正規化してから検証
- 表示用の4桁区切り形式を入力された場合も、ハイフン除去で同一キーとして扱う

### 4.2 形態からの情報取得

クライアントは形態コードをパースして以下を判定可能:

- **種別**: P1/P2 → 買い切り
- **利用可能メジャーバージョン**: P1→1, P2→2（形態コードの `01` / `02` に対応）

---

## 5. 後方互換

既存の `PDFH-` + 32文字形式（例: `PDFH-A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6`）は、移行期間中は有効として扱う。サーバー側で形式を判定し、旧形式の場合は `purchased` かつ `purchased_version` 未設定として扱う（初回アクティベーション時に `purchased_version` を設定）。

---

## 6. 生成元

| 生成元 | 形態コード |
|--------|------------|
| Stripe 決済（買い切り・`checkout.session.completed`） | P1 + 現在のメジャーバージョン（環境変数で指定、デフォルト01） |
| 手動発行（Supabase） | P101, P102 等（買い切りと同じルール） |

---

## 7. 環境変数

| 変数名 | 説明 | デフォルト |
|--------|------|------------|
| `LICENSE_PURCHASED_MAJOR_VERSION` | 買い切り発行時のメジャーバージョン（1, 2...） | 1 |

## 8. 手動発行時のキー生成

Supabase で手動発行する場合、Stripe Webhook や発行スクリプトと同様に HMAC 署名を付与するロジックを使用すること。

- **シリアル部**: 暗号論的に安全な乱数から28文字（16進大文字）
- **HMAC署名**: `PDFH:P101:{シリアル部}` を `LICENSE_SECRET_KEY` で HMAC-SHA256 署名し、必要な桁数を付加
