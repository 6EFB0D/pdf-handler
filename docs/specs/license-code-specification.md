# ライセンスコード仕様書

## 1. 形式

```
PDFH-<ライセンス形態4文字>-<28文字>
```

- **プレフィックス**: `PDFH-`（固定）
- **ライセンス形態**: 4文字（種別2文字 + バージョンコード2文字）
- **シリアル部**: 28文字（英数字、大文字）

**例**: `PDFH-P101-A1B2C3D4E5F6G7H8I9J0K1L2M3`

---

## 2. ライセンス形態（4文字）

### 2.1 種別（先頭2文字）

| コード | 意味 |
|--------|------|
| P1 | 買い切り（Purchased）v1 |
| P2 | 買い切り v2（将来用） |
| S1 | サブスクリプション Standard |
| S2 | サブスクリプション Premium |

### 2.2 バージョンコード（後ろ2文字）

| コード | 意味 |
|--------|------|
| 01 | メジャーバージョン 1（買い切り v1.x 用） |
| 02 | メジャーバージョン 2（買い切り v2.x 用、将来） |
| 00 | 全バージョン（サブスクリプション用） |

### 2.3 形態コード一覧

| 4文字 | 種別 | バージョン制限 |
|-------|------|----------------|
| P101 | 買い切り v1 | v1.x まで |
| P102 | 買い切り v2 | v2.x まで（将来） |
| S100 | サブスク Standard | 全バージョン |
| S200 | サブスク Premium | 全バージョン |

---

## 3. シリアル部（28文字）

- **文字種**: 0-9, A-F（16進数大文字）
- **生成**: 暗号論的に安全な乱数から生成
- **一意性**: 重複しないこと（UUID ベースまたは十分なエントロピー）

---

## 4. バリデーション

### 4.1 形式チェック

- 正規表現: `^PDFH-[P1P2S1S2][0-9]{2}-[0-9A-F]{28}$`
- ハイフンは2つのみ（プレフィックス後、形態後）

### 4.2 形態からの情報取得

クライアントは形態コードをパースして以下を判定可能:

- **種別**: P1/P2 → 買い切り、S1/S2 → サブスク
- **利用可能メジャーバージョン**: P1→1, P2→2, S1/S2→全バージョン（00）

---

## 5. 後方互換

既存の `PDFH-` + 32文字形式（例: `PDFH-A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6`）は、移行期間中は有効として扱う。サーバー側で形式を判定し、旧形式の場合は `purchased` かつ `purchased_version` 未設定として扱う（初回アクティベーション時に `purchased_version` を設定）。

---

## 6. 生成元

| 生成元 | 形態コード |
|--------|------------|
| Stripe 決済（買い切り） | P1 + 現在のメジャーバージョン（環境変数で指定、デフォルト01） |
| Stripe 決済（サブスク Standard） | S100 |
| Stripe 決済（サブスク Premium） | S200 |
| 手動発行（Supabase） | 用途に応じて P101, S100, S200 等 |

---

## 7. 環境変数

| 変数名 | 説明 | デフォルト |
|--------|------|------------|
| `LICENSE_PURCHASED_MAJOR_VERSION` | 買い切り発行時のメジャーバージョン（1, 2...） | 1 |

## 8. 手動発行時のキー生成

Supabase で手動発行する場合、以下の形式でキーを生成:

```sql
-- 買い切り v1 の例
SELECT 'PDFH-P101-' || UPPER(SUBSTRING(REPLACE(gen_random_uuid()::TEXT, '-', ''), 1, 28));

-- サブスク Standard の例
SELECT 'PDFH-S100-' || UPPER(SUBSTRING(REPLACE(gen_random_uuid()::TEXT, '-', ''), 1, 28));
```
