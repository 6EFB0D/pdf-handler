# 3. 領収証の発行を求められたときの処理（企業向け・個人情報保護）

## 概要

企業顧客から領収証を求められた場合の対応方法です。**個人情報を公開しない**ための運用を前提にしています。

## 推奨: Stripe の領収証を活用

Stripe は決済完了時に**領収証を自動送信**する機能があります。個人の情報を一切使わずに済みます。

### Stripe 領収証の有効化

1. [Stripe ダッシュボード](https://dashboard.stripe.com/settings/billing/automatic) を開く
2. 「Receipts」セクションを開く
3. 以下を設定:
   - **Customer emails**: オン（決済完了時に顧客へメール送信）
   - **Receipt template**: 必要に応じてカスタマイズ（ロゴ、会社名など）

### 領収証に表示される内容

- 決済日時
- 金額
- 商品名（Stripe の Product/Price で設定）
- 領収証番号
- 閲覧用リンク（30日間有効）

### 個人情報を出さないポイント

- **会社名**: Stripe の「Business settings」で「Business name」を設定（例: 「PDFハンドラ」や「事業者名」）
- **住所**: 必要なら事業用の住所を設定（自宅住所は避ける）
- **メール**: Stripe の送信元は `receipts@stripe.com` など Stripe のドメイン。あなたの個人メールは使われない

---

## 企業向けの追加対応

### 1. Stripe の設定を企業向けに整える

**Stripe ダッシュボード** → Settings → Business settings:

| 項目 | 設定例 | 個人情報保護 |
|------|--------|--------------|
| Business name | PDFハンドラ / あなたの事業者名 | 個人名は使わない |
| Business address | 事業用住所（任意） | 自宅住所は避ける |
| Support email | support@yourdomain.com | 個人メールは使わない |

### 2. 領収証のカスタマイズ

Stripe ダッシュボード → Settings → Branding:

- **Icon / Logo**: アプリのアイコンや事業者ロゴ
- **Brand color**: ブランドカラー

これにより、領収証が「Stripe の汎用レシート」ではなく、事業者としての体裁を保てます。

### 3. 顧客が「領収証が届かない」と言った場合

1. **迷惑メールフォルダ**を確認してもらう
2. Stripe の領収証リンクを再送:
   - Stripe ダッシュボード → Payments → 該当決済 → 「Resend receipt」
3. それでも届かない場合は、領収証の URL をコピーしてメールで送付
   - Stripe の決済詳細ページに「View receipt」リンクがある

## 手動で領収証を発行する場合

Stripe の自動領収証で足りない場合（例: 企業の経理フォーマットに合わせたい）は、手動発行を検討します。

### 個人情報を出さないポイント

| 項目 | 推奨 | 避ける |
|------|------|--------|
| 発行者名 | 事業者名・屋号 | 個人の氏名 |
| 連絡先 | support@yourdomain.com | 個人メール |
| 住所 | 事業用住所（任意） | 自宅住所 |
| 印鑑 | 不要（電子領収証なら） | 実印・個人印 |

### 領収証のテンプレート例

```
領収証

発行日: 2025年2月10日
領収証番号: RCP-20250210-001

【お支払いいただいた方】
（顧客が入力した会社名・氏名）

【金額】
金 3,480円（税込）

【但し書き】
PDFハンドラ Standard版（買い切り） ライセンス

【発行元】
発行元: PDFハンドラ（または事業者名）
お問い合わせ: support@yourdomain.com
```

- 発行元は個人名ではなく事業者名のみ
- 連絡先はサポート用メールのみ

### 手動発行の運用

1. 顧客から領収証を依頼（フォームまたは support@ へ）
2. Stripe で決済を確認
3. 上記テンプレートで PDF を作成
4. support@ から顧客へ送付（個人メールは使わない）

---

## 問い合わせフォームで個人情報を守る

領収証の依頼を「フォーム」で受け、個人のメールアドレスを出さない方法:

1. **Google フォーム** や **Formspree** などでフォームを作成
2. フォームの送信先を `support@yourdomain.com` に転送
3. フォームの「送信完了」メッセージで「数営業日以内にご連絡します」と案内
4. 顧客への返信は `support@` から行う

これにより、顧客には「support@」しか見えず、個人のメールアドレスは一切公開されません。
