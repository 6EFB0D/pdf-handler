# リリースビルド手順書

## 方法1: Visual Studio（推奨・確実）

### 手順

1. **Visual Studioでプロジェクトを開く**
   - `PdfHandler.sln` をダブルクリック

2. **Releaseモードに変更**
   - 上部ツールバーで「Debug」→「Release」に変更
   - プラットフォームは「Any CPU」のまま

3. **ソリューションのリビルド**
   - メニュー: `ビルド` → `ソリューションのリビルド`
   - 完了まで待つ（1-2分）

4. **ビルド結果の確認**
   - 出力ウィンドウで「ビルド: 成功 X 個」を確認
   - エラーがないことを確認

5. **実行ファイルの場所**
   ```
   src\PdfHandler.UI\bin\Release\net8.0-windows\
   ```

6. **配布フォルダを作成**
   - デスクトップなどに新しいフォルダを作成
   - 名前: `PdfHandler_v3.4.2`

7. **ファイルをコピー**
   - 手順5のフォルダ内のすべてのファイルを
   - 手順6のフォルダにコピー

8. **README.txtを追加**
   - 修正版README.txtをコピー

9. **ZIPファイルを作成**
   - フォルダを右クリック
   - `送る` → `圧縮（zip形式）フォルダー`

10. **完成！**
    - `PdfHandler_v3.4.2.zip` が完成

## 方法2: コマンドライン（dotnet publish）

### 前提条件
- .NET 8.0 SDK がインストールされている
- コマンドプロンプトまたはPowerShellを使用

### 手順

1. **コマンドプロンプトを開く**
   - プロジェクトフォルダで右クリック
   - 「ターミナルで開く」または「コマンドプロンプトで開く」

2. **ビルドコマンドを実行**
   ```cmd
   dotnet publish src\PdfHandler.UI\PdfHandler.UI.csproj -c Release -r win-x64 --self-contained true
   ```

3. **出力先を確認**
   ```
   src\PdfHandler.UI\bin\Release\net8.0-windows\win-x64\publish\
   ```

4. **配布フォルダを作成してコピー**
   - 上記フォルダの内容をすべてコピー

5. **README.txtを追加**

6. **ZIPファイルを作成**

---

## 方法3: バッチファイル（自動）

### ⚠️ 文字化け問題の解決

元のバッチファイルは文字コードの問題で失敗します。
以下の2つの新しいバッチファイルを用意しました。

### ファイル1: build_release_en.bat
- 英語メッセージ版
- 文字化けなし
- UTF-8対応

### ファイル2: build_release_simple.bat
- シンプル版
- 最小限のメッセージ
- 文字化けリスク最小

### 使い方

1. **バッチファイルをプロジェクトフォルダに配置**
   - `PdfHandler.sln` と同じフォルダ

2. **README.txtも同じフォルダに配置**

3. **ダブルクリックで実行**
   - `build_release_simple.bat` をダブルクリック

4. **完了を待つ**
   - 自動で以下が作成される:
     - `PdfHandler_v3.4.2\` フォルダ
     - `PdfHandler_v3.4.2.zip` ファイル

## トラブルシューティング

### Q1: 「dotnetが認識されない」エラー

**原因**: .NET SDK がインストールされていない

**解決策**:
1. https://dotnet.microsoft.com/download からダウンロード
2. .NET 8.0 SDK をインストール
3. PCを再起動

### Q2: バッチファイルが文字化けする

**原因**: 文字コードの問題

**解決策**:
- `build_release_simple.bat` を使用
- または Visual Studio から手動ビルド

### Q3: 「アクセスが拒否されました」エラー

**原因**: ファイルが使用中、または権限不足

**解決策**:
1. Visual Studio を閉じる
2. アプリが実行中なら終了
3. 管理者として実行

### Q4: ZIPファイルが作成されない

**原因**: PowerShellの実行ポリシー制限

**解決策**:
手動でZIPを作成:
1. フォルダを右クリック
2. `送る` → `圧縮（zip形式）フォルダー`

## ビルド後の確認

### 必須チェック項目

1. **実行ファイルの確認**
   - [ ] PdfHandler.UI.exe が存在する
   - [ ] ダブルクリックで起動する

2. **DLLファイルの確認**
   - [ ] PdfHandler.Core.dll
   - [ ] PdfHandler.Infrastructure.dll
   - [ ] itext関連のDLL
   - [ ] Docnet関連のDLL

3. **動作確認**
   - [ ] フォルダを開ける
   - [ ] PDFが表示される
   - [ ] 結合・分割が動作する

4. **配布ファイルの確認**
   - [ ] README.txt が含まれている
   - [ ] ZIPファイルのサイズが妥当（100-200MB程度）

## バージョン番号の変更方法（次回リリース時）

### 1. バッチファイルを編集
```bat
set VERSION=3.4.3  ← ここを変更
```

### 2. README.txtを編集
```
バージョン: 3.4.3  ← ここを変更
リリース日: 2024年12月XX日  ← ここを変更
```

### 3. CHANGELOG.mdに追記
新しいバージョンの変更内容を記載

## 配布時の注意事項

1. **ウイルススキャン**
   - Windows Defenderでスキャン
   - VirusTotalでチェック（オプション）

2. **動作確認**
   - 別のPCでテスト（可能なら）
   - クリーンなWindows環境で確認

3. **ファイルサイズの確認**
   - 異常に大きい/小さくないか
   - 通常: 100-200MB程度

4. **配布方法の選択**
   - OneDrive/Google Drive
   - メール添付（サイズ制限注意）
   - USBメモリ
   - 社内サーバー
