# インライン編集機能ガイド

## 🎯 F2キーでのファイル名変更

リストビューで直接ファイル名を編集できる機能を追加しました！

### 使い方

#### 方法1: F2キーで編集開始
1. リストビューでPDFファイルを選択
2. **F2キー**を押す
3. ファイル名が編集可能になります
4. 新しいファイル名を入力
5. **Enterキー**で確定、または**Escキー**でキャンセル

#### 方法2: ダブルクリックで編集開始
1. リストビューでファイル名を**ダブルクリック**
2. ファイル名が編集可能になります
3. 新しいファイル名を入力
4. **Enterキー**で確定、または**Escキー**でキャンセル

### 自動処理

- **.pdf拡張子の自動追加**: 拡張子を入力しなくても自動的に`.pdf`が追加されます
- **不正な文字のチェック**: ファイル名に使用できない文字が含まれている場合、エラーメッセージが表示されます
- **重複チェック**: 同じファイル名が既に存在する場合、システムがエラーを表示します

### キーボードショートカット

| キー | 動作 |
|------|------|
| F2 | 編集開始 |
| Enter | 変更を確定 |
| Esc | 変更をキャンセル |
| Tab | 次のフィールドへ移動 |

### 注意事項

- **サムネイルビューでは使用できません** - リストビューに切り替えてください
- **編集中は他の操作ができません** - Enterまたはescで編集を終了してください
- **フォーカスが外れると自動確定** - 他の場所をクリックすると変更が確定されます

## 🔧 技術的な変更点

### 追加されたプロパティ（PdfFileInfo）
```csharp
[ObservableProperty]
private bool _isEditing;  // 編集モードフラグ

[ObservableProperty]
private string _editingName = string.Empty;  // 編集中の名前
```

### XAMLの変更
- ファイル名列を`TextBlock`と`TextBox`の切り替え表示に変更
- バインディングで`IsEditing`プロパティに基づいて表示/非表示を制御

### イベントハンドラー
- `FileListView_KeyDown` - F2キー検出
- `FileName_DoubleClick` - ダブルクリック検出
- `FileNameEdit_KeyDown` - Enter/Escキー処理
- `FileNameEdit_LostFocus` - フォーカス喪失時の自動確定
- `FileNameEdit_Loaded` - 編集開始時のフォーカスと全選択

## 🎨 ユーザーエクスペリエンス

### 従来の方法との比較

#### 従来（ダイアログ方式）
1. ファイルを選択
2. メニューまたはF2キー
3. **ダイアログが開く** ← 画面切り替え
4. ファイル名を入力
5. OKボタンをクリック
6. ダイアログが閉じる

#### 新方式（インライン編集）
1. ファイルを選択
2. F2キー
3. **その場で編集** ← シームレス
4. Enterキー
5. 完了

### メリット
- ✅ 画面の切り替えなし
- ✅ マウス操作が不要
- ✅ 素早い編集が可能
- ✅ Excelのようなスムーズな操作感
