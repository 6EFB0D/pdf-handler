**作成日**: 2025年1月1日  
**対象**: Week 3-4

---

## 1. ページ抽出機能

### 1.1 概要
PDFファイルから指定したページを抽出し、新しいPDFファイルとして保存する機能。

### 1.2 機能要件

#### 基本機能
- 単一ページの抽出
- 複数ページの抽出（連続ページ範囲）
- 複数ページの抽出（非連続ページ）
- 抽出先ファイル名の指定
- 抽出先フォルダの指定

#### UI要件
- 抽出ダイアログの表示
- ページ範囲の入力（例: "1-5,10,15-20"）
- プレビュー表示（抽出対象ページの確認）
- 進捗表示

#### 技術要件
- PdfSharpを使用したページ抽出
- ファイルロック回避（メモリストリーム方式）
- エラーハンドリング（無効なページ番号、範囲外等）

### 1.3 ユーザーストーリー
```
ユーザーは、PDFファイルから特定のページを抽出して、
新しいPDFファイルとして保存したい。
```

### 1.4 受け入れ基準
- [ ] 単一ページの抽出が正常に動作する
- [ ] 連続ページ範囲の抽出が正常に動作する
- [ ] 非連続ページの抽出が正常に動作する
- [ ] 無効なページ番号が入力された場合、適切なエラーメッセージが表示される
- [ ] 抽出先ファイル名が正しく設定される
- [ ] ファイルロックが発生しない

## 2. ページ回転機能

### 2.1 概要
PDFファイルの指定したページを回転させる機能。

### 2.2 機能要件

#### 基本機能
- 単一ページの回転（90度、180度、270度）
- 複数ページの回転（選択したページ範囲）
- 全ページの回転
- 回転方向の選択（時計回り、反時計回り）

#### UI要件
- 回転ボタン/メニューの表示
- 回転角度の選択（90度、180度、270度）
- 回転対象ページの選択
- プレビュー表示（回転後の確認）

#### 技術要件
- PdfSharpを使用したページ回転
- 元ファイルの上書き保存または新規保存
- ファイルロック回避

### 2.3 ユーザーストーリー
```
ユーザーは、PDFファイルのページを回転させて、
正しい向きに修正したい。
```

### 2.4 受け入れ基準
- [ ] 単一ページの回転が正常に動作する
- [ ] 複数ページの回転が正常に動作する
- [ ] 全ページの回転が正常に動作する
- [ ] 回転角度が正しく適用される
- [ ] 元ファイルが正しく更新される（または新規ファイルが作成される）

## 3. ページ削除機能

### 3.1 概要
PDFファイルから指定したページを削除する機能。

### 3.2 機能要件

#### 基本機能
- 単一ページの削除
- 複数ページの削除（連続ページ範囲）
- 複数ページの削除（非連続ページ）
- 削除確認ダイアログの表示

#### UI要件
- 削除ボタン/メニューの表示
- 削除対象ページの選択
- 削除確認ダイアログ
- プレビュー表示（削除後のページ構成の確認）

#### 技術要件
- PdfSharpを使用したページ削除
- 元ファイルの上書き保存または新規保存
- ファイルロック回避
- 削除後のページ番号の再割り当て

### 3.3 ユーザーストーリー
```
ユーザーは、PDFファイルから不要なページを削除して、
ファイルサイズを削減したい。
```

### 3.4 受け入れ基準
- [ ] 単一ページの削除が正常に動作する
- [ ] 連続ページ範囲の削除が正常に動作する
- [ ] 非連続ページの削除が正常に動作する
- [ ] 削除確認ダイアログが表示される
- [ ] 削除後のページ番号が正しく再割り当てされる
- [ ] 元ファイルが正しく更新される（または新規ファイルが作成される）

## 4. 共通要件

### 4.1 エラーハンドリング
- 無効なページ番号の入力
- 範囲外のページ番号の指定
- ファイルアクセス権限エラー
- ディスク容量不足エラー

### 4.2 パフォーマンス
- 大容量PDFファイル（100MB以上）の処理
- 100ページ以上のPDFファイルの処理
- メモリ使用量の最適化

### 4.3 ユーザー体験
- 進捗表示（処理中のフィードバック）
- エラーメッセージの明確化
- 操作の取り消し（Undo）機能（将来実装）

3. **中優先度**: ページ削除機能

## 6. 技術的考慮事項

### 6.1 PdfSharpの使用
- PdfSharp 6.1.1を使用
- Importモードでのページ操作
- メモリ効率の良い実装

### 6.2 ファイルロック回避
- メモリストリーム方式を使用
- 元ファイルを即座に解放
- プレビュー中でも操作可能

### 6.3 UI/UX設計
- MVVMパターンに従う
- 既存のUIパターンと一貫性を保つ
- アクセシビリティを考慮
