### 2.2 BYOK（Bring Your Own Key）モデルの詳細

#### 仕組み

```
【従来の統合モデル】
ユーザー → PDFハンドラ → PDFハンドラのAPIキー → Claude API
                      ↓
                   APIコストは
                   PDFハンドラが負担

【BYOKモデル】
ユーザー → PDFハンドラ → ユーザーのAPIキー → Claude API
                      ↓
                   APIコストは
                   ユーザーが直接負担

メリット:
✅ ユーザー: 使用量に応じた公正なコスト
✅ PDFハンドラ: APIコスト負担なし
✅ 透明性: ユーザーがコストを完全把握
```

#### 実装方法

```csharp
// 設定画面
public class AISettingsViewModel
{
    // ユーザーの選択
    public enum AIKeyMode
    {
        PDFHandlerManaged,  // オプション1: PDFハンドラ管理
        BYOK                // オプション2: ユーザー自身のキー
    }

    public AIKeyMode SelectedMode { get; set; }
    
    // BYOKの場合
    public string ClaudeApiKey { get; set; }
    public string OpenAIApiKey { get; set; }
    public string AzureVisionKey { get; set; }
}

// AI機能呼び出し時
public async Task<string> SummarizeDocument(byte[] pdfBytes)
{
    if (Settings.SelectedMode == AIKeyMode.BYOK)
    {
        // ユーザー自身のAPIキーを使用
        var client = new AnthropicClient(Settings.ClaudeApiKey);
        // コストはユーザーのAnthropicアカウントに請求
    }
    else
    {
        // PDFハンドラのAPIキーを使用
        var client = new AnthropicClient(_managedApiKey);
        // 使用量をカウント、月次請求
    }
    
    return await client.SummarizeAsync(pdfBytes);
}
```

## 3. 各AI APIの取得方法（ユーザーガイド）

### 3.1 Claude API（Anthropic）

```
【取得手順】

1. Anthropic Console にアクセス
   https://console.anthropic.com

2. アカウント作成
   - メールアドレス
   - パスワード
   - 電話番号（SMS認証）

3. 支払い方法登録
   - クレジットカード
   - または従量課金アカウント

4. APIキー生成
   - Settings → API Keys
   - "Create Key"
   - キーをコピー（一度しか表示されない）

5. PDFハンドラに設定
   - 設定 → AI機能 → Claude APIキー
   - 貼り付けて保存

【料金】
- 初回クレジット: $5（約¥750）無料
- その後従量課金
- 入力: $3/1M tokens
- 出力: $15/1M tokens

【予算設定】
- Usage Limits で月間上限設定可能
- 例: $10/月 → 約300回の要約
```

### 3.2 OpenAI API（セマンティック検索用）

```
【取得手順】

1. OpenAI Platform にアクセス
   https://platform.openai.com

2. アカウント作成
   - メールアドレス
   - Google/Microsoft アカウントでも可

3. 支払い方法登録
   - クレジットカード
   - 初回$5クレジット付与

4. APIキー生成
   - API keys → Create new secret key
   - キーをコピー

5. PDFハンドラに設定
   - 設定 → AI機能 → OpenAI APIキー
   - 貼り付けて保存

【料金】
- text-embedding-3-small: $0.02/1M tokens
- 非常に安価（検索1回 = ¥0.001程度）

【予算設定】
- Usage limits で上限設定

### 3.3 Azure Computer Vision（OCR用、オプション）

```
【取得手順】

1. Azure Portal にアクセス
   https://portal.azure.com

2. Computer Vision リソース作成
   - リソースの作成 → AI + Machine Learning
   - Computer Vision を選択

3. 設定
   - 価格レベル: Free（月間5,000トランザクション）
   - または Standard（$1.50/1,000トランザクション）

   - 設定 → AI機能 → Azure Vision
   - キーとエンドポイントを入力

【料金】
- Free tier: 月間5,000トランザクション
- Standard: $1.50/1,000トランザクション
- OCR 1ページ = 1トランザクション

【代替案】
- TesseractOCR（完全無料、ローカル）
- PDFハンドラに標準搭載予定
- 精度: Azure 95% vs Tesseract 85%

## 5. セキュリティとプライバシー

### 5.1 APIキーの保存方法

```csharp
// セキュアな保存（暗号化）
public class SecureKeyStorage
{
    // Windows Data Protection APIを使用
    public void SaveApiKey(string keyName, string apiKey)
    {
        // ユーザープロファイルで暗号化
        byte[] encryptedData = ProtectedData.Protect(
            Encoding.UTF8.GetBytes(apiKey),
            null,
            DataProtectionScope.CurrentUser
        );
        
        // レジストリまたはローカルファイルに保存
        // 他のユーザーやアプリからは読み取り不可
        SaveEncryptedData(keyName, encryptedData);
    }
    
    public string LoadApiKey(string keyName)
    {
        byte[] encryptedData = LoadEncryptedData(keyName);
        byte[] decryptedData = ProtectedData.Unprotect(
            encryptedData,
            null,
            DataProtectionScope.CurrentUser
        );
        return Encoding.UTF8.GetString(decryptedData);
    }
}
```

### 5.2 データプライバシー

```
【BYOKモデルの利点】

1. データの流れ
   ユーザーのPC → ユーザーのAPIキー → Claude API
   
   → PDFハンドラのサーバーを経由しない
   → データがPDFハンドラに保存されない
   → プライバシー保護

2. Anthropic/OpenAIのデータポリシー
   - ユーザーのデータはモデル訓練に使用されない
   - APIリクエストは30日後に削除
   - GDPR、CCPA準拠

3. ユーザー通知
   「あなたのAPIキーを使用するため、PDFの内容は
    Claude APIに送信されますが、PDFハンドラの
    サーバーには保存されません。」
```
