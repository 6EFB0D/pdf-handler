# PDFハンドラ 実行手順書

## アプリケーションの起動方法

### 方法1: Visual Studioから実行（推奨）

#### 1. スタートアッププロジェクトの確認

1. ソリューションエクスプローラーで **PdfHandler.UI** を右クリック
2. **「スタートアップ プロジェクトに設定」** を選択
3. PdfHandler.UI が太字になっていることを確認

#### 2. 実行

**デバッグ実行:**
```
F5 キーを押す
または
メニュー: デバッグ → デバッグの開始
```

**デバッグなしで実行:**
```
Ctrl + F5 キーを押す
または
メニュー: デバッグ → デバッグなしで開始
```

### 方法2: コマンドラインから実行

```bash
# プロジェクトディレクトリに移動
cd pdf-handler\src\PdfHandler.UI

# 実行
dotnet run
```

### 方法3: 実行ファイルから直接起動

```bash
# ビルド後の実行ファイルを起動
cd pdf-handler\src\PdfHandler.UI\bin\Debug\net6.0-windows
PdfHandler.UI.exe
```

## トラブルシューティング

### 問題1: ウィンドウがすぐに閉じる

**原因:**
- App.xamlの設定が間違っている
- 起動時にエラーが発生している

**解決方法:**

1. **Visual Studioで実行する**
   - F5（デバッグ実行）で実行
   - 出力ウィンドウやエラーリストを確認

2. **エラーダイアログを確認**
   - 修正版では起動エラーがダイアログ表示されます
   - エラーメッセージを確認

3. **App.xamlを確認**
   - `StartupUri` が削除されていることを確認
   - 以下のようになっているはず:
   ```xml
   <Application x:Class="PdfHandler.UI.App"
                xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
                xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
                xmlns:converters="clr-namespace:PdfHandler.UI.Converters">
   ```

### 問題2: 「PdfHandler.UI.exeを開始できません」

**原因:** ビルドされていない

**解決方法:**
```bash
dotnet build
```

### 問題3: DLLが見つからないエラー

**原因:** 依存DLLが不足

**解決方法:**
```bash
# クリーンビルド
dotnet clean
dotnet restore
dotnet build
```

## アプリケーションの使い方

### 初回起動時

1. **アプリケーションが起動します**
   - 3ペイン構成のウィンドウが表示されます

2. **フォルダを選択**
   - メニュー: `ファイル` → `フォルダを開く`
   - または ツールバーの 📁 ボタン
   - PDFファイルが含まれるフォルダを選択

3. **PDFファイルの確認**
   - 左ペイン: フォルダツリー
   - 中央ペイン: PDFファイル一覧（サムネイル/リスト）
   - 右ペイン: プレビュー（ON/OFF可能）

### 主要機能

#### 1. 表示切替

- **サムネイル/リストビュー切替**
  - ツールバーの 🖼️ または 📋 ボタン
  - または メニュー: `表示` → `サムネイルビュー` / `リストビュー`

- **プレビューON/OFF**
  - ツールバーの 👁️ ボタン
  - または メニュー: `表示` → `プレビュー表示`
  - または `F12` キー

#### 2. ファイル操作

- **ファイル名変更**
  - ファイルを選択して、ツールバーの ✏️ ボタン
  - または `F2` キー
  - ※現在は「実装中」メッセージが表示されます

- **ファイル削除**
  - ファイルを選択して、ツールバーの 🗑️ ボタン
  - または `Delete` キー

#### 3. PDF操作

- **PDF結合**
  - 複数のPDFを選択（Ctrl + クリック）
  - ツールバーの 🔗 ボタン
  - または メニュー: `ツール` → `PDFを結合`
  - または `Ctrl + M`

- **PDF分割**
  - PDFを選択
  - ツールバーの ✂️ ボタン
  - または メニュー: `ツール` → `PDFを分割`
  - または `Ctrl + Shift + S`

## 開発者向け: デバッグ実行

### Visual Studioでのデバッグ

1. **ブレークポイントを設定**
   - コードの行番号左側をクリック

2. **デバッグ実行**
   - F5 キーを押す

3. **ステップ実行**
   - F10: ステップオーバー
   - F11: ステップイン
   - Shift + F11: ステップアウト

### デバッグ情報の確認

#### 出力ウィンドウ
- メニュー: `表示` → `出力`
- または `Ctrl + Alt + O`

#### エラーリスト
- メニュー: `表示` → `エラーリスト`
- または `Ctrl + \, E`

#### イミディエイトウィンドウ
- メニュー: `デバッグ` → `ウィンドウ` → `イミディエイト`
- または `Ctrl + Alt + I`

## 既知の制限事項

現在の実装では以下の機能が制限されています：

### プレースホルダー実装
- **PDFプレビュー**: ページ番号とファイル名のみ表示
- **サムネイル**: PDFアイコンのみ表示
- **ファイル名変更**: ダイアログ未実装（メッセージのみ）

### 完全動作機能
- ✅ フォルダツリー表示
- ✅ ファイル一覧表示
- ✅ PDFページ数取得
- ✅ PDF結合（iText7）
- ✅ PDF分割（iText7）
- ✅ 表示切替機能

## パフォーマンスメモ

### 推奨環境
- メモリ: 4GB以上
- ストレージ: 100MB以上の空き容量
- CPU: デュアルコア以上

### 大量ファイル処理
- 1フォルダあたり1000ファイル程度まで快適に動作
- それ以上の場合、読み込みに時間がかかる場合があります
